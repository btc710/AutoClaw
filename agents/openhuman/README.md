# agents/openhuman/

OpenHuman adapter for the AutoClaw agent router. OpenHuman is published by Tinyhumans.ai.

In AutoClaw, OpenHuman is the natural home for the *persona* and *embodiment* dimensions of the copilot: tone, mood, voice, gesture vocabulary, and continuity of character across drives.

Responsibilities:

- Hosting persona state and feeding it back into the router so other agents can write “in character”.
- Mapping motion/emotion outputs to the character renderer’s animation states.
- Exposing voice-profile selection (calm, hype, deadpan, etc.) and persisting the driver’s choice locally.
